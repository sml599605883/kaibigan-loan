# WebView Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a secure in-app WebView and replaceable H5-to-native bridge to Kaibigan Loan.

**Architecture:** `modules/webview` separates protocol parsing, action dispatch, and browser UI. Routes and `NavigationHelper` open HTTP(S) targets inside the container; the dispatcher calls existing native services and returns stable bridge results. The reserved account-change action returns an explicit unavailable result until its native selection flow exists.

**Tech Stack:** Flutter, GetX, flutter_inappwebview 6.1.5, flutter_test.

```dart
class WebViewBridgeResult {
  const WebViewBridgeResult({required this.code, required this.message, this.data = const {}});
  factory WebViewBridgeResult.success([Map<String, dynamic> data = const {}]) =>
      WebViewBridgeResult(code: 0, message: 'success', data: data);
  factory WebViewBridgeResult.failure(String message, {int code = -1}) =>
      WebViewBridgeResult(code: code, message: message);
  final int code;
  final String message;
  final Map<String, dynamic> data;
  String encode() => jsonEncode({'code': code, 'message': message, 'data': data});
}
```

Malformed input returns empty values. Define a temporary random handler and
nine semantic action constants: risk, external browser, scheme, close, home,
review, common params, retry, and reserved account change.

static Future<T?>? toWebView<T extends Object?>({required String url, String? title}) {
  final uri = _resolveWebUri(url);
  if (uri == null) return null;
  return Get.toNamed<T>(AppRoutes.webView, arguments: {'url': uri.toString(), 'title': title});
}
